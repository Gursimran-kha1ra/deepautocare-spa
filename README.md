# Deep Auto Care — Single-Page Site (SPA)

A clean, contact-focused one-page site for **Deep Auto Care**, driven by `config.json`.

## Page sections
Hero → Services → Why choose us → 24/7 roadside → Gallery (carousel) →
Before/After → Contact. (Reviews and FAQ sections were intentionally removed.)

- **Simple service cards** — icon, name, one line, and a **Contact us** button. No
  pricing. Tapping a card scrolls to the contact form and pre-selects that service.
- **Gallery is a partial-view carousel** — current photo centered, neighbours peeking;
  swipe, arrows, or dots.
- Contact-first: header/hero CTA is **Contact us**; the contact form replies by
  phone/text/email.

## Responsive
Tuned for phones, tablets, laptops and large/ultra-wide monitors (the content column
widens on big screens instead of sitting in a narrow band). See `css/spa.css`.

## SEO & agentic (AI-agent) visibility
Generated by `node tools/seo-build.js` from `config.json`:

- **`index.html`** ships a full static `<head>`: title, description, canonical,
  hreflang, `robots` (with `max-image-preview:large`), Open Graph, Twitter cards, geo
  tags, a rich **JSON-LD `@graph`** (Organization, WebSite, AutoRepair/LocalBusiness,
  WebPage, BreadcrumbList, and a Service node per service), and a **`<noscript>`
  fallback** — so search crawlers *and* AI agents can read the business without running
  JavaScript.
- **`robots.txt`** allows all crawlers and explicitly welcomes AI answer-engines
  (GPTBot, ClaudeBot, PerplexityBot, Google-Extended, Applebot-Extended, CCBot, …).
- **`sitemap.xml`**, **`site.webmanifest`**, and **`llms.txt`** (a Markdown brief for
  LLM agents following the emerging `/llms.txt` convention).

Re-run `node tools/seo-build.js` after editing `config.json` to regenerate all of the
above.

## Run it
Serve over HTTP (loads `config.json` via `fetch`, so `file://` won't work):
```bash
cd automech-spa-simple
python3 -m http.server 8080     # open http://localhost:8080
```

```
├── index.html         ← generated: static SEO head + noscript + mounts

├── css/main.css  css/spa.css
├── js/config.js  components.js  gallery.js  booking.js  spa.js

```
